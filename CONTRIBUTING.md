# Contributing to Banking Fraud Platform

Thank you for your interest in contributing! This repository is owned by Caleb-Muinde83 and is intended to be maintained by the broader Datech Community.

## How to Contribute

1. Fork the repository.
2. Create a new branch for your work (`feature/<name>` or `fix/<name>`).
3. Make your changes with clear commit messages.
4. Push your branch to your fork.
5. Open a pull request against `Caleb-Muinde83/banking-fraud-platform`.

## Pull Request Guidelines

- Include a clear description of what changed and why.
- Reference related issues when applicable.
- Keep changes focused and avoid large, unrelated edits.
- Ensure the code is formatted consistently and that any new dependencies are documented.

## Issue Reporting

If you find a bug or want to request a feature:

- Open an issue with a descriptive title.
- Provide steps to reproduce the issue.
- Include relevant environment details, expected behavior, and actual behavior.

## Review and Approval

- Community maintainers are encouraged to review pull requests promptly.
- Approved maintainers can merge PRs once the changes are validated.
- If you are not sure whether a change is ready, request review from a maintainer.

## Code of Conduct

This project follows a respectful, inclusive, and constructive code of conduct. Please be polite, patient, and collaborative.

## Maintainers and Collaborators

The Datech Community may be granted write/admin access for ongoing maintenance and review.

If you are interested in joining as a maintainer, please reach out to Caleb or a current community maintainer.
